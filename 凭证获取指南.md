# AI 费用监控系统 — 凭证获取指南

> 本文档说明 `.env` 中各项凭证的获取方式。除 MySQL 外，其他凭证**按需配置**——缺失则对应平台显示"未配置"，不影响系统启动。

## 1️⃣ DeepSeek API Key

**申请地址**：https://platform.deepseek.com/api_keys

**获取步骤**：
1. 登录 DeepSeek 开放平台（如无账号先注册 + 实名）
2. 顶部菜单 → 「API Keys」
3. 点击「创建 API Key」，复制生成的 `sk-xxxxxxxx`
4. 粘贴到 `.env`：
   ```
   DEEPSEEK_API_KEY=sk-你的key
   ```

**官方文档**：https://api-docs.deepseek.com/

---

## 2️⃣ Kimi / Moonshot API Key

**申请地址**：https://platform.moonshot.cn/console/api-keys

**获取步骤**：
1. 登录 Moonshot 开放平台
2. 左侧菜单 → 「API Key 管理」
3. 点击「创建 API Key」，复制 `sk-xxxxxxxx`
4. 粘贴到 `.env`：
   ```
   MOONSHOT_API_KEY=sk-你的key
   ```

**官方文档**：https://platform.moonshot.cn/docs

---

## 3️⃣ 火山引擎 AK/SK

**申请地址**：https://console.volcengine.com/iam/keymanage/

**获取步骤**：
1. 登录火山引擎控制台（需实名认证）
2. 右上角头像 → 「访问密钥」 → 「新建密钥」
3. 复制 **Access Key ID (AK)** 和 **Secret Access Key (SK)**
4. 粘贴到 `.env`：
   ```
   VOLC_AK=AKxxxxxx
   VOLC_SK=SKxxxxxx
   ```
5. ⚠️ 还需确保账号已开通「火山方舟」或「视觉智能」等用到余额的服务

**官方文档**：https://www.volcengine.com/docs/6369/1625875（IAM 密钥管理）

## 4️⃣ OpenAI-Hub Token + UserID

**申请地址**：https://api.openai-hub.com/

**获取步骤**：
1. 登录 OpenAI-Hub 控制台
2. 进入「个人中心」或「令牌管理」页面
3. 复制 **Token**（访问令牌）
4. 记下 **User ID**（用户 ID，通常是数字）
5. 粘贴到 `.env`：
   ```
   OPENAIHUB_BASE_URL=https://api.openai-hub.com
   OPENAIHUB_TOKEN=你的token
   OPENAIHUB_USER_ID=你的userid
   ```

> 该平台为第三方代理，UI 可能调整，关键是从「个人中心/账户设置」页找到 Token 和 User ID。

## 5️⃣ 智谱AI Cookie（抓包方式）

⚠️ 智谱AI 没有官方余额查询 API，需登录控制台后抓包获取。

**申请/登录地址**：https://open.bigmodel.cn/

**获取步骤**：
1. 浏览器登录 https://open.bigmodel.cn/（建议 Chrome）
2. 按 `F12` 打开开发者工具 → 切到 **Network** 标签
3. 刷新「财务中心」或「账户中心」页面（让余额数字加载出来）
4. 在 Network 列表里找返回余额数据的 XHR 请求（通常是 `XHR` 类型，URL 含 `balance` / `finance` / `account`）
5. 右键该请求 → 「Copy」 → 「Copy as cURL (bash)」
6. 从 cURL 中提取：
   - **请求 URL** → 填 `ZHIPU_BALANCE_URL`
   - **Cookie 请求头** → 填 `ZHIPU_COOKIE`
   - 在响应 JSON 中找到余额字段路径 → 填 `ZHIPU_BALANCE_FIELD`（如 `data.balance`）
   - 如果有币种字段 → 填 `ZHIPU_CURRENCY_FIELD`
7. 粘贴到 `.env`：
   ```
   ZHIPU_COOKIE=你的完整cookie串
   ZHIPU_BALANCE_URL=https://open.bigmodel.cn/api/xxx/balance
   ZHIPU_BALANCE_FIELD=data.balance
   ZHIPU_CURRENCY_FIELD=data.currency
   ```

**备选：Coding Plan 配额模式**（如果用的是 Coding 套餐）：
```
ZHIPU_TOKEN=你的token
ZHIPU_QUOTA_URL=https://open.bigmodel.cn/api/monitor/usage/quota/limit
```

> ⚠️ Cookie 一般 7-30 天过期，过期后需重新抓包。

## 6️⃣ MiniMax Cookie（抓包方式）

⚠️ MiniMax 同样无官方余额 API，需抓包。

**申请/登录地址**：https://platform.minimaxi.com/

**获取步骤**：
1. 浏览器登录 https://platform.minimaxi.com/
2. `F12` → **Network** 标签
3. 进入「财务中心」或「费用账单」页面
4. 找返回余额的 XHR 请求（URL 含 `balance` / `finance` / `wallet`）
5. 右键 → 「Copy as cURL (bash)」
6. 提取 URL、Cookie、余额字段路径
7. 粘贴到 `.env`：
   ```
   MINIMAX_COOKIE=你的完整cookie串
   MINIMAX_BALANCE_URL=https://platform.minimaxi.com/api/xxx/balance
   MINIMAX_BALANCE_FIELD=data.balance
   MINIMAX_CURRENCY_FIELD=data.currency
   ```

> ⚠️ Cookie 过期需重新抓包。

## 7️⃣ SendCloud 邮件配置

**申请地址**：https://www.sendcloud.net/

**获取步骤**：
1. 注册 SendCloud 账号（免费额度 50 封/天）
2. 登录后进入「邮件」 → 「API_USER」页面
3. 创建 API_USER，记录：
   - **API_USER**（如 `HRflag_monitor`）
   - **API_KEY**（密钥）
4. 配置发信域名（需在 DNS 添加 SPF/DKIM 记录，SendCloud 会引导）
5. 创建邮件模板「模型API余额预警」，记录 **模板调用名**
6. 粘贴到 `.env`：
   ```
   SENDCLOUD_API_USER=你的api_user
   SENDCLOUD_API_KEY=你的api_key
   SENDCLOUD_FROM_EMAIL=monitor@yourdomain.com
   SENDCLOUD_FROM_NAME=AI费用监控
   SENDCLOUD_MAIL_TEMPLATE=model_api_balance_warning
   ```

**官方文档**：https://www.sendcloud.net/doc/

## 8️⃣ SendCloud 短信配置

**申请地址**：https://www.sendcloud.net/ （同账号下）

**获取步骤**：
1. SendCloud 控制台 → 「短信」 → 「签名管理」 → 添加签名（如「AI费用监控」，需审核）
2. 「短信」 → 「模板管理」 → 创建模板（如「您的【AI费用监控】余额低于阈值，当前余额 X 元」），审核通过后记录 **模板 ID**
3. 「短信」 → 「API_USER」 → 创建短信专用 API_USER，记录 **SMS_USER** 和 **SMS_KEY**
4. 粘贴到 `.env`：
   ```
   SENDCLOUD_SMS_USER=你的sms_user
   SENDCLOUD_SMS_KEY=你的sms_key
   SENDCLOUD_SMS_TEMPLATE_ID=模板ID
   SENDCLOUD_SMS_SIGN=AI费用监控
   ```

```
ALERT_EMAIL_TO=zhangsan@hrflag.com,lisi@hrflag.com

```

## 🔔 告警阈值（可选调整）

默认阈值可在 `.env` 调整，也可启动后在 Web UI 的「系统设置」页面修改（会覆盖 `.env`）：
```
YELLOW_THRESHOLD=500   # 余额<500元 → 黄色告警
RED_THRESHOLD=200      # 余额<200元 → 红色告警
RED_REPEAT_HOURS=6     # 红色告警每6小时重复一次
```

## ✅ 启动验证

凭证填好后，启动服务：
```bash
npm start
```
访问 http://localhost:3000 ，应看到六宫格监控面板。已配置的平台会显示余额，未配置的显示「未配置」。

## 🆘 常见问题

| 问题 | 解决 |
|------|------|
| 抓包 Cookie 失效 | 重新登录控制台抓包，更新 `.env` |
| SendCloud 发信失败 | 检查域名 SPF/DKIM 是否配置、API_USER 状态是否「启用」 |
| 火山引擎 AK/SK 403 | 确认账号已开通对应产品（如火山方舟） |
| 阿里云 RDS 连不上 | 检查白名单是否加入出口 IP `218.1.208.36` |
